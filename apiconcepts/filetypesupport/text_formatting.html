    Text Formatting

    Complex documents often contain formatting and various styles. For example, most editors will support character formats such as Bold, Italic and Underline.

    The purpose of semi-WYSIWYG formatting support is to allow for any framework editor (i.e. an editor built on top of the framework) to display some level of character formatting information based on common formatting types supported by the framework. Thus, the formatting supported by a specific external editor such as Microsoft Word will be converted by a native processor object (at parseing time) into the framework's semi-WYSIWYG formatting objects. These semi-WYSIWYG formatting objects of the framework can be persisted to a bilingual file format, as indeed they are, for the framework XLIFF file format and read back in by the framework when opening such a document.

    To apply character formatting, the Sdl.FileTypeSupport.Framework.Formatting class is used. This class contains basic formatting properties which can be used in native or bilingual filters.

    Formatting properties should be always applied to IStartTagProperties and you can assign multiple IFormattingItem items to one Formatting property. The text will receive all the formatting that is defined in the opening tag.

    IStartTagProperties and IEndTagProperties have CanHide properties that allows user to hide tags. In order not to have a tag displayed in the editor, set the CanHide property to True.
