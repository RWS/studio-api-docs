Class Framework1ExternalPreviewException

This exception is thrown by the TTX filter framework 1 preview writer adapter (TtxPreviewNativeWriter) when a FF 1 filter has successfully generated an external preview, which it will manage itself (as opposed to the preview that is managed by a preview application or preview control).

Remarks

Explicitly handling this exception allows a preview host application to be notified that the preview has been successfully generated, but should be treated as a "fire and forget" preview. The preview files typically are not the file names that are passed to the writer, and they will be automatically deleted when the external preview application is closed by the user.

Note that throwing an exception here is rather a hack, however for the moment no a better way of handling this situation is available.
